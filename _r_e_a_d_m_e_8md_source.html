11 [**Sentry**](https://sentry.io/welcome/) is such a useful tool, but there is currently [no official support](https://sentry.io/platforms/) for C++. **Crow** is an official Sentry client for C++ that tries to fill this gap to allow for Sentry notifications in long-running C++ applications where you do not want to constantly look at log files.

13 ## API
14 
15 ### Setup
16 
17 - `crow::crow(dsn, context=null, install_handlers=true)` to create a client
18 - `crow::install_handler()` to later install termination handler
19 
20 ### Reporting
21 
22 - `crow::capture_message(message, context=nullptr, async=true)` to send a message
23 - `crow::capture_exception(exception, context=nullptr, async=true, handled=true)` to send an exception
24 - `crow::add_breadcrumb(message, type="default")` to add a breadcrumb
25 - `crow::get_last_event_id()` to get the id of the last event
26 
27 ### Context management
28 
29 - `crow::get_context()` to return current context
30 - `crow::add_user_context(const json& data)` to add data to the user context
31 - `crow::add_tags_context(const json& data)` to add data to the tags context
32 - `crow::add_request_context(const json& data)` to add data to the request context
33 - `crow::add_extra_context(const json& data)` to add data to the extra context
34 - `crow::merge_context(const json& context)` to merge context information
35 - `crow::clear_context()` to reset context
36 
37 See [the documentation](https://nlohmann.github.io/crow/classnlohmann_1_1crow.html) for a complete overview of the public API.

39 ## Requirements for Sentry SDKs
40 
41 The following items from [the SDK implementation guidelines](https://docs.sentry.io/clientdev/overview/#writing-an-sdk) are realized at the moment:
42 
43 - The following items are expected of production-ready SDKs:
44  - [x] DSN configuration
45  - [ ] Graceful failures (e.g. Sentry server is unreachable)
46  - [x] Setting attributes (e.g. tags and extra data)
47  - [x] Support for Linux, Windows and OS X (where applicable)
48 - Feature based support is required for the following:
49  - [ ] If cookie data is available, it’s not sent by default
50  - [ ] If POST data is available, it’s not sent by default
51 - Additionally, the following features are highly encouraged:
52  - [x] Automated error capturing (e.g. uncaught exception handlers)
53  - [ ] Logging framework integration
54  - [x] Non-blocking event submission
55  - [ ] Basic data sanitization (e.g. filtering out values that look like passwords)
56  - [x] Context data helpers (e.g. setting the current user, recording breadcrumbs)
57  - [ ] Event sampling
58  - [ ] Honor Sentry’s HTTP 429 Retry-After header
59  - [ ] Pre and Post event send hooks
60  - [ ] Local variable values in stacktrace (on platforms where this is possible)

89 ## Prerequisites
90 
91 You need [libcurl](https://curl.haxx.se/libcurl/), [zlib](https://zlib.net), and a C++11 compiler to use the library.
92 Curl should be detected by CMake. If this does not work, you can download [libcurl](https://curl.haxx.se/download.html)
93 and [zlib](https://zlib.net) and pass the path to the source release folder to CMake via
94 `-DCROW_EXTERNAL_CURL_PROJECT=curl-7.61.0` and `-DCROW_EXTERNAL_ZLIB_PROJECT=zlib-1.2.11`.
95 This libcurl and zlib is then built and linked.
